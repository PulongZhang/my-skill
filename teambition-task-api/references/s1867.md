## 写前核对

### 更新已有任务

先用 `POST /work-time-server/api/tasks/bulk` 查询明确任务 ID，并逐项核对：

| 核对项 | 判定 |
|---|---|
| 项目 | 任务实际 `_projectId` 为 `695b09c7b842a4a0fd053603` |
| SFC | 任务实际 `_scenariofieldconfigId` 与目标任务类型一致 |
| 可操作性 | `isDeleted=false` 且 `isArchived=false` |
| 当前状态 | 核对 `isDone` 和 `_taskflowstatusId`，确认确实需要变更 |

### 创建新任务

任务尚不存在，不能先查任务详情。写入前应在循环外一次性查询以下元数据并确认配置存在；创建后还必须独立校验任务列表、阶段、执行者、项目、SFC、起始状态及删除/归档/完成标志：

| 核对项 | 查询或确认方式 |
|---|---|
| 任务列表 | `GET /api/tasklists?_projectId={projectId}`，确认 `_tasklistId` 属于当前项目 |
| 阶段 | `GET /api/stages?_tasklistId={tasklistId}`，确认 `_stageId` 属于该任务列表；该接口返回阶段，不是工作流状态 |
| SFC | `GET /api/v2/projects/{projectId}/scenariofieldconfigs`，确认 `_scenariofieldconfigId` 是目标任务类型 |
| 起始状态 | 当前仅验证需求 SFC 的 `904`。其他 SFC 必须由用户提供并确认 `_taskflowstatusId` 属于实际 SFC 工作流且 `kind: start`，否则停止创建；`GET /api/stages` 不是工作流状态列表 |

本页不记录一次性任务 ID、工时记录 ID、临时查询结果或浏览器会话数据。

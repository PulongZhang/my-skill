# 钉钉会议 AI 听记输入模板

用于从钉钉会议 AI 听记导出的“纪要 Markdown”和“文字记录 Markdown”中抽取关键信息。生成标准会议纪要前，先按下列模板整理可确认信息；缺失或无法判断的字段留空或写“待确认”，不要编造。

## 纪要 Markdown 关键信息模板

适用于以 `# 纪要` 开头，包含主题、时间、参与人、会议背景、分章节摘要、待办或总结的导出文件。

```markdown
### 纪要文档关键信息

- 来源文件：
- 会议主题：
- 会议时间：
- 参与人：
- 主持人/主讲人：
- 会议背景/目的：
- 主要讨论内容：
  1. 
  2. 
  3. 
- 已确认结论：
  1. 
- 问题记录候选：
  | 提问者 | 问题内容 | 跟进者 | 解决方案 | 结果 |
  | --- | --- | --- | --- | --- |
  |  |  |  |  |  |
- 遗留问题候选：
  | 问题描述 | 跟进者 | 预期完成时间 |
  | --- | --- | --- |
  |  |  |  |
- 不应保留内容：图片链接、AI 洞察、章节时间线、重复待办、评价性描述。
```

## 文字记录 Markdown 关键信息模板

适用于包含创建时间、转写时长、关键词和 `## **转文字**` 的导出文件，正文通常按“发言人 + 时间戳 + 发言内容”排列。

```markdown
### 文字记录关键信息

- 来源文件：
- 创建时间：
- 转写时长：
- 关键词：
- 可推断会议主题：
- 可推断主持人/主讲人：
- 明确提出的问题：
  | 时间 | 提问者 | 问题内容 | 回应人 | 回应/处理方式 |
  | --- | --- | --- | --- | --- |
  |  |  |  |  |  |
- 明确确认的结论或决策：
  | 时间 | 发言人 | 结论/决策 |
  | --- | --- | --- |
  |  |  |  |
- 明确分配的行动项：
  | 时间 | 事项 | 负责人 | 完成时间/条件 |
  | --- | --- | --- | --- |
  |  |  |  |  |
- 需要与纪要文档互相校验的点：
  1. 
- 应删除的转写噪声：口头禅、寒暄、重复确认、明显识别错误、无关片段。
```

## 合并规则

- 同时提供纪要文档和文字记录文档时，以纪要文档作为结构骨架，以文字记录补充提问者、责任人、解决方案、状态和原始上下文。
- 纪要文档与文字记录冲突时，优先采用文字记录中有明确发言依据的内容；仍无法判断则写“待确认”。
- 不把上述模板原样输出到最终纪要；模板只用于中间抽取，最终仍按 `standard-format.md` 输出。
